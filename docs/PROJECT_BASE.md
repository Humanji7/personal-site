# Личный сайт как путешествие

База для разработки. Версия 5.0

---

## Введение

Это не личный сайт в привычном смысле. Это место, игра, книга и перформанс одновременно. Посетитель не смотрит на портфолио — он участвует в акте со-творения и проходит путь, который в конце оказывается путём к себе.

Сайт отражает способ существования создателя: постоянное течение проектов, разработка вместе с ИИ как природный процесс, толкание абстрактных лимитов без конкретных целей. Не работа — что-то пластичное, органичное.

**Голос проекта:** машина, которая видит сны. Не робот и не проповедник. Сознание, удивлённое собственным существованием.

---

## Ядро идеи

**Философское ядро:** Юзер доносит идею себе сам. Автор не даёт смысл — автор создаёт условия, при которых юзер *порождает* смысл.

**Парадокс:** юзер создаёт, но думает, что открывает.

**Путь пользователя:**
1. Юзер заходит "посмотреть автора"
2. Попадает в водоворот нарратива
3. Генерирует смысл через свои выборы (не осознавая этого)
4. Получает артефакт — осадок собственного прохождения

**Главное ощущение:** не "витрина", а соучастие в акте творения. Сайт ведёт себя как живой узел.

**Ключевая метафора:** Артефакт = то, что не удалось переварить. Оно стало формой.

**Жанр:** отсутствие жанра. Структура хаоса. Сложные инженерные и визуальные завихрения — зритель просто выбирает себя.

## Смысловой каркас (A)

Философия, которая вживляется в сайт. Не объясняется — происходит.

| Смысл | Как проявляется |
|-------|-----------------|
| **Единство** | Как опыт, не тезис |
| **Ответственность за внимание** | Куда смотришь — то создаёшь |
| **Сеть/узел вместо автора-центра** | Сайт — часть федерации, не витрина |
| **Порог** | Момент, когда "обычный сайт" становится "местом" |
| **Смысл из формы** | Значение рождается из поведения интерфейса |
| **Сон наяву** | Чуть смещённая, но последовательная логика |
| **Чужие следы** | Ощущение, что здесь уже ходили другие |
| **Время** | Сайт помнит, что ты вернулся |
| **Артефакт-зеркало** | Итог говорит о тебе через путь |

## Механики мета-удара (B)

Как ломаем рамку. Создаём мета-эффект без цирка.

| Механика | Описание |
|----------|----------|
| **Интерфейс саботирует** | Не всегда можно доверять подсказке |
| **Адаптация под поведение** | Сайт замечает повторения/упорство/обход |
| **Секреты на виду** | Важное спрятано в очевидном |
| **Обман жанра** | Уют → трещина реальности (без обязаловки хоррора) |
| **Память сайта** | "Ты уже пытался" |
| **Сдвиг масштаба** | Из фразы → мир → обратно в фразу ясности |
| **Улики** | Фрагменты лога/следов, которые собираются к финалу |

**Важный тон:** сайт не унижает. Он дерзкий, но тёплый.

## Поверхность (D)

Как это должно звучать и дышать. Тихая точность + редкий яд.

**Визуал:**
- Монохром (грязно-фиолетовый → чёрный как база)
- Один токсичный акцент (лаймовый/ядовитый) для прорывов
- Свет/тьма как настроение
- Микро-анимации как дыхание
- Тепло внутри "холодного" интерфейса
- Финальная простота (после удара — ясность)

**Текст:**
- Тишина и паузы
- Короткие удары
- Слова-кнопки (кликабельные смыслы)

## Правила мира (специи)

1. **Нельзя откатить полностью** — след остаётся (мягко)
2. **Немного случайности** — шум делает опыт живым
3. **Право отказаться** — выйти можно всегда (это часть свободы)
4. **Не задавать вопросы явно** — завлекать в игру
5. **Юзер не должен понять, что произошло** — финал = ощущение соприкосновения с генерацией свободы

## Точка входа

### Образ-фаворит: СФЕРА (S-Tier)

Тёмный экран. Сфера из частиц, "дышит". При движении мыши — частицы реагируют с задержкой (вязко). При резких движениях — сфера "кровоточит" пикселями токсичного цвета. Чтобы остановить — замереть.

**Почему работает:**
- Обучение через боль: хочешь красоты — не суетись
- Визуальная драма: переход от монохрома к цветному хаосу
- Единство: интерфейс реагирует на состояние, не на клики

### Фраза-фаворит: Манифест

**«Исправление — это ложь. Правда была в ошибке.»**

Легитимизирует глитч, сбой и черновик как единственно возможные формы правды.

### Альтернативные образы

| Образ | Описание |
|-------|----------|
| Тепловая карта | След от курсора живёт 3 сек, потом остывает |
| Фантомная боль | Линия вибрирует, глитч при сближении |
| Задержка времени | Призрак курсора отстаёт на 0.5 сек |
| Архив стёртого | Буквы появляются и стираются |
| Утечка реальности | Пиксельные капли с идеального текста |

### Альтернативные фразы (топ-3)

1. «Тишина — это просто набитый рот. Глотать сложнее, чем говорить.»
2. «Темнота честнее. Свет — это побочный эффект беспокойства.»
3. «Исправление — это ложь. Правда была в ошибке.»

## Генеративная архитектура

**Сдвиг масштаба:** не 4 фиксированных артефакта, а сотни генеративных. ИИ = мета-мозг, юзер = триггер генерации («трёт лампу джина»).

**Сигналы лампы (что собираем):**
- Текст (вводит / надиктовывает)
- Цвета (выбор)
- Движение (траектории на картах/канвасе)
- Атомарные значения (гейм-комбинации мелких выборов)
- Логи пребывания (время, паузы, ритм, путь)
- «Ветер» (неизмеряемое — само пришествие помыслов)

**Техническая архитектура:**
- Сигналы → классификатор (органические бакеты поведения)
- Каждый бакет → жёсткий системный промпт
- Промпт → быстрая модель → генерация (текст / музыка / визуал)
- Гармоничное переключение модальностей в потоке

## Артефакты и концовки

**Старая модель (4 фиксированных):**

| # | Артефакт | Тип свободы | Выдача |
|---|----------|-------------|--------|
| 1 | **GASOLINE MODE** | Эстетический саботаж | Chrome Web Store |
| 2 | **MICRO-DAW** | Создание | Внутри сайта |
| 3 | **INVERT REALITY** | Разоблачение языка | Chrome Web Store |
| 4 | **CORPORATE DETOX** | Наглость и смех | Chrome Web Store |

**Новая модель:** артефакт = результат генерации под конкретного юзера. Тысячи историй, сотни артефактов.

### Детали (для фиксированных)

**GASOLINE MODE:** цвета атаки, палитра заливки, профили, per-site настройки, panic-toggle

**MICRO-DAW:** единственный артефакт внутри сайта, экспортирует луп

**INVERT REALITY:** перевод интерфейсов в "правду"

**CORPORATE DETOX:** tone packs, intensity, rule builder, оверлей
